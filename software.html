Installing TriDAS

In order to be able to modify any of the code that generates the odmbdev page, one first needs to check out TriDAS from CVS. Note that XDAQ is required to actually run the odmbdev script, so it is usually preferable to check it out into a directory on wimp.physics.ucsb.edu. After logging in to wimp.physics.ucsb.edu, create a personal directory with "mkdir my_dir_name" and switch into it with "cd my_dir_name". Put the script initial_setup.sh in this directory. In it, replace all instances of "username" and "my_dir_name" with appropriate values. Running this script will check out and compile the TriDAS repository for the first time. To pull down updates from CVS, use the script get_updates.sh. This, too, should be placed in the directory "my_dir_name" and you will need to make the appropriate replacements of "username" and "my_dir_name" in the script before using.

Generating the odmbdev Page

After checking out the repository (see above), one can edit the code in the TriDAS/emu/odmbdev directory. After making any changes, you will need to recompile the code with "compile.sh", found in TriDAS/emu. Once this is done, you are ready to produce the odmbdev page. First, to prevent conflicts, use "ps aux | grep xdaq" to check that nobody else is running the script. If not, go into the odmbdev directory (note, the following step only works if you are in the odmbdev directory!) and enter the command "./script/startME11Dev.sh". This will generate the odmbdev page.
